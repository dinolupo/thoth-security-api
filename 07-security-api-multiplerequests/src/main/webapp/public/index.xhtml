If request 'A' sets the Principal, can I make requests 'B', 'C', and 'D' and maintain that Principal across the requests?

Yes!

In the EE 8 Security API, the @AutoApplySession annotation on the HttpAuthenticationMechanism is what does this for you. Once a Principal is established, @AutoApplySession prevents HttpAuthenticationMechanism#validateRequest() from being called again.
