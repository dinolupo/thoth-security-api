When I get to the important code (the code I write to create the Principal and groups) can I use CDI to get an instance of some object which will help me get the principal and group information?

Yes!

EE Security is a 1st class CDI citizen. So CDI can definately be used.

EE Security defines an IdentityStore just for this purpose. In this example, TestHttpAuthenticationMechanism uses...

@Inject
private IdentityStoreHandler identityStoreHandler;

...to inject an IdentityStoreHandler. The IdentityStoreHandler is an EE Security class designed to help you validate principal and get the role information. IdentityStoreHandler delegates this job to the IdentityStore implementations, and you may have more than 1 implementation to do different things (like contact different sources to get roles).
