When the Security API is being used by a WAR, will it protect only the WAR it's in or every WAR deployed to the server?

Only this WAR!

JASPIC is really low-level and depending on how you register the AuthConfigProvider with the AuthConfigFactory, you can protect just the WAR the JASPIC code is in, every WAR deployed to the server, or one of the WARs deployed to the server. The Security API, however, doesn't work this way.

By default, when the Security API is used by a WAR, the Security API protects only that WAR. Unlike JASPIC, there is no way (as of this writing August, 2017) to configure the Security API to protect other other WARs on the server.
